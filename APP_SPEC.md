## 1. アプリ概要

### コンセプト
Apple Music／CD取り込み音源を含む **全リスニング履歴を分析・可視化** するiOSアプリ。  
Apple Music の公式 Replay では見られない「CDを含む本当の自分の音楽履歴」を記録・分析することが核心価値。

### 技術スタック
| 項目 | 内容 |
|------|------|
| OS | iOS 18+ |
| UI | SwiftUI ネイティブ |
| データ取得 | MediaPlayer framework（playCount / lastPlayedDate / isLocalAsset） |
| ローカルDB | Core Data（App Group 経由でWidget共有） |
| クラウド | iCloud 同期（CloudSyncService） |
| Widget | WidgetKit（3種類） |
| Live Activity | NowPlaying Live Activity（Dynamic Island 対応） |
| アートワーク | Deezer API / iTunes Search API + カスタム上書き |
| ハプティクス | HapticsManager（light / medium / success / error） |

## 3. データモデルとデータ取得の制約

### Apple Music の制約

Apple Music の `MPMediaItem` から取得できる情報は以下のみ:
```
playCount       — 累計再生回数（タイムスタンプなし）
lastPlayedDate  — 最終再生日時（1件のみ）
isLocalAsset    — CD取り込み曲かどうか（assetURL の有無）
```

**取得できないもの:**
- 個別の再生タイムスタンプ（いつ再生したか）
- 再生デバイス / 再生モード
- Apple Music のサーバー側Replay データ

### 同期アルゴリズム（PlayHistoryTracker）

```
初回起動
└─ performInitialSnapshotOnly()
   ─ PlayCountSnapshotEntity のみ保存（PlayHistory は生成しない）
   ─ hasInitialSnapshot = true

以降の同期
└─ performIncrementalSyncBatched()
   ─ 前回スナップショット vs 現在 playCount を比較
   ─ diff > 0 の曲に PlayHistoryEntity を生成（playedAt は lastPlayedDate から逆算）
   ─ PlayCountSnapshotEntity を更新
   ─ diffSyncCount += 1
```

### HistoryAccuracyLevel

| レベル | 条件 | 意味 |
|---|---|---|
| baseline | diffSyncCount = 0 | 初回スナップショットのみ |
| early | 1〜4回 | データ蓄積初期 |
| developing | 5〜14回 | 分析に使えるが誤差あり |
| established | 15回以上 | 高精度 |

### Core Data エンティティ

| エンティティ | 主要フィールド |
|---|---|
| PlayHistoryEntity | trackID / title / artistName / albumTitle / playedAt / playCountSnapshot / duration / isLocalAsset |
| PlayCountSnapshotEntity | trackID / playCount / recordedAt |
| FavoriteEntity | trackID / title / artistName / favoritedAt |
| SearchHistoryEntity | query / searchedAt |
| CustomImageEntity | key / imageData / updatedAt |

## 4. 機能改善案

### 4-1. CDアイコン統一削除【優先度: 高】

**現状の課題**: `opticaldisc` アイコンが以下に残存している。ランキング画面では削除済みだが、他画面で不統一。

| ファイル | 箇所 |
|---|---|
| `HomeView.swift` | `HomeRankingRow` の楽曲タイトル横 |
| `TrackDetailView.swift` | タイトル横 |
| `LibraryView.swift` | `TrackRowView` の楽曲タイトル横 |

**提案**: ライブラリ・楽曲詳細・ホームからも削除。ただし「統計 > 音源の内訳」のアイコンとしての使用（CDのシンボル）は維持する。

---

### 4-2. ライブラリを主要タブへ昇格【優先度: 中】

**現状**: ライブラリは `more > ライブラリ` と2タップ必要。  
**課題**: 楽曲検索・閲覧はアプリのコア操作なのに深い階層に埋まっている。

**提案案 A（タブ5つ）**:
```
ホーム / ライブラリ / ランキング / 月別 / more
```

**提案案 B（タブ4つ、月別を more へ移動）**:
```
ホーム / ランキング / ライブラリ / more（月別・年別・パーソナリティ…）
```

---

### 4-3. 月別・年別レポートのアートワーク表示【優先度: 中】

**現状**: TOP楽曲リストにアートワークがなく、テキストのみ。  
**提案**: `TrackArtworkView(track:, size: 40)` を追加して視覚的豊かさを向上。

---

### 4-4. ホーム画面のサマリからCDアイコンを廃止【優先度: 低】

**現状**: リスニングサマリの「CD取り込み曲」に `opticaldisc` アイコンを使用。  
**提案**: `internaldrive` や `music.note` に変更してアイコンをモダン化。またこのスタット自体を「ローカル音源 x%」のように比率表示に変える。

---

### 4-5. 初回起動時の empty state 改善【優先度: 高】

**現状**: 初回起動直後は月別・年別レポートが "データはまだありません" の空状態のみ。ユーザーは何をすればデータが増えるか不明。  
**提案**: baseline / early 状態専用の説明画面を追加。

```
[スナップショット取得済み]
あなたのライブラリを記録しました。
次回以降の同期でリスニング履歴が蓄積されます。
今後 X 回の同期で詳細分析が始まります。
```

---

### 4-6. アーティスト詳細のアルバム一覧【優先度: 中】

**現状**: `ArtistDetailView` にアルバム一覧の表示がない（確認要）。  
**提案**: アーティスト配下のアルバム一覧を折りたたみ可能セクションで追加。

---

### 4-7. 検索・ランキングの期間連動【優先度: 低】

**現状**: ランキング検索中は期間フィルターが非表示になり、全件から検索する。  
**提案**: 検索中も期間フィルターを維持し、「7日間でこの曲は何回?」を検索から確認できるようにする。

---

### 4-8. more タブのタイトル大文字化【優先度: 低】

**現状**: `navigationTitle("more")` と小文字。他タブと不統一。  
**提案**: `"その他"` または `"More"` に統一。

### 5-3. 月別レポートのリデザイン

**現状課題**:
- 月セレクターが単純な `←/→` ボタン — スワイプで切り替えられない
- サマリカードの情報が縦に列挙されているだけ
- ジャンル分析が地味

**提案**:

1. **スワイプ月切り替え**: ScrollView or TabView(page) でスワイプ対応
2. **ハイライトカード**: その月の「特別な瞬間」（最も聴いた日 + その曲）を大きく表示
3. **ジャンル可視化**: 円グラフ（PieChart）か、ドーナツチャートで割合を直感的に表示
4. **アートワーク付きTOPトラック**: 楽曲にアートワーク（44px）を追加

### 5-4. ランキング画面のリデザイン

**現状課題**:
- リスト形式のみ。TOP3が視覚的に強調されているが、絵文字だけ
- 期間切り替えは Menu ピッカーで発見しにくい

**提案**:

1. **TOP3 ポディウム**: 1位を中央に大きく、2位左・3位右でポディウム形式
2. **期間フィルター**: SegmentedPicker を直接表示（Menu より発見しやすい）
3. **バー付きリスト**: 各行に再生数のミニバーを追加し、相対的な差を可視化

```
┌──────────────────────────────────────┐
│    🥈 2位      🥇 1位     🥉 3位    │  ← ポディウム
│  [art 56px] [art 72px] [art 56px]   │
│   アーティスト  アーティスト  …     │
│    ○○○回      ○○○回     ○○○回   │
├──────────────────────────────────────┤
│ 4. タイトル         ▮▮▮▮▮▮▮ 234回 │  ← バー付き行
│ 5. タイトル         ▮▮▮▮▮▮ 198回  │
└──────────────────────────────────────┘
```

### 5-5. パーソナリティ画面の追加要素

現在の実装は良いベースがあるため、以下を追加提案。

**① パーソナリティ変遷タイムライン**
月ごとのパーソナリティをタイムラインで表示。「3ヶ月前は音楽探検家だったのに今はヘビロテ職人になっている」といった変化を可視化。

**③ パーソナリティ獲得バッジ**

### 5-10. アクセシビリティ

**現状**: `@Environment(\.accessibilityReduceMotion)` はパーソナリティ画面で対応済み。他画面は未対応。

**追加すべき対応**:

| 項目 | 対応方法 |
|---|---|
| ReduceMotion | ホーム・月別・年別でもアニメーションを無効化 |
| Dynamic Type | 楽曲タイトルの最大フォントサイズ制限を外す |
| VoiceOver | ランキング行に `accessibilityLabel("1位, Lemon, 米津玄師, 234回再生")` |
| 高コントラスト | グラジェント上の白テキストにシャドウを強化 |

## 6. 新機能アイデア

### 6-1. パーソナリティ進化タイムライン

毎月のパーソナリティを CoreData に保存し、「自分の音楽人格の変遷」を時系列グラフで可視化。

```swift
// PersonalityHistoryEntity (新規)
// month: Int, year: Int, personalityRawValue: String, score: Double
```

---

### 6-2. リスニングストリーク

連続して音楽を聴いた日数を記録。「30日連続リスニング 🔥」のようなモチベーション演出。

---

### 6-3. スマートインサイト通知

「先週より再生数が20%増えました」「新しいジャンルを開拓しています」のような週次通知。既存の `NotificationService` を拡張で対応可能。

---

### 6-4. ジャンルドリルダウン

統計のジャンルバーをタップ → そのジャンルのTOPアーティスト・楽曲を表示する専用ビュー。

---

### 6-5. 楽曲メモ

楽曲詳細に1〜3行のメモを保存できる機能。「このライブで初めて聴いた」「○年○月に○○で」など、音楽体験の記録。CoreData に `NoteEntity` 追加で対応可能。

---

### 6-6. 比較モード

「今月 vs 先月」「今年 vs 去年」を並べて比較するビュー。月別・年別レポートに追加。

---

### 6-7. Album of the Year / Track of the Year

年間レポートで「今年最も感情を揺さぶった1枚」を自動または手動で選定し、特別なカードとしてシェア。

---

### 6-8. ライブラリ変化レポート

「先月から追加した楽曲」「最近聴かなくなった楽曲」をリスト表示。音楽ライフスタイルの変化を客観視できる。
